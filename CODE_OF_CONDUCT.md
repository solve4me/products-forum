# Código de Conduta - Comunidade Produtos

Este documento estabelece as diretrizes de convivência e operação para o fórum público de **Produtos**. Como este repositório é dedicado exclusivamente à interação da comunidade através do *GitHub Discussions*, nosso objetivo é garantir um ambiente técnico, seguro, colaborativo e produtivo para a troca de ideias, resolução de dúvidas e coleta de feedback.

---

## 🎯 Nosso Compromisso

A equipe de produtos se compromete a promover uma comunidade aberta e profissional. Garantimos que a participação no nosso fórum seja uma experiência livre de assédio para todos, independentemente de nível de experiência técnica, identidade de gênero, orientação sexual, deficiência, aparência pessoal, raça, etnia, idade ou religião.

---

## ⚖️ Padrões de Comportamento

Para manter o rigor técnico e a alta qualidade das discussões, exigimos que todos os membros sigam os padrões abaixo.

### Comportamentos Esperados

* **Comunicação Clara e Objetiva:** Ao relatar um comportamento inesperado ou propor uma ideia, forneça contexto técnico suficiente, logs (sanitizados) e cenários de reprodução.
* **Foco na Solução:** Ofereça críticas construtivas. Se discordar de uma abordagem arquitetural ou de negócios, embase seu argumento tecnicamente.
* **Colaboração e Empatia:** Utilize linguagem profissional e acolhedora. Ajude outros membros respondendo a dúvidas caso você tenha experiência no tópico abordado.

### Comportamentos Inaceitáveis

* **Assédio e Toxidez:** Comentários insultuosos, depreciativos, provocações (*trolling*) ou ataques pessoais.
* **Spam e Ruído Comercial:** Divulgação não solicitada de serviços de terceiros, links irrelevantes ou conteúdo fora do escopo do ecossistema Solve4ME.
* **Exposição de Terceiros (Doxxing):** Publicar informações privadas ou de contato de outros membros da comunidade sem permissão explícita.

## 📌 Diretrizes Operacionais do Fórum

Como nosso repositório opera estritamente como um fórum (*Discussions*), o cumprimento das regras operacionais abaixo é obrigatório para manter o repositório organizado:

| Diretriz | Regra de Aplicação |
| :--- | :--- |
| **Pesquise Antes de Criar** | Utilize a busca do GitHub Discussions. Evite criar tópicos duplicados para problemas ou ideias já mapeadas. |
| **Sanitização de Dados Críticos** | **NUNCA** publique dados reais de produção. Mascare obrigatoriamente informações como `Tokens (JWT/HMAC)`, `Client Secrets`, `Telefones (PII)`, `URLs sensíveis` ou `IDs de Organização` (Ex: utilize `[REDACTED]`). |
| **Categorização Estrita** | Publique sua postagem na categoria exata correspondente (ex: *Ideas*, *Q&A*, *General*). Tópicos fora de contexto serão movidos ou encerrados pela moderação. |
| **Uso de Upvotes vs. "Bumps"** | Não adicione comentários vazios como "+1", "Up" ou "Alguém?". Utilize a função nativa de **Upvote** do GitHub para impulsionar a relevância de um tópico. |

## 🛡️ Moderação e Reporte de Incidentes

A equipe de mantenedores possui o direito e a responsabilidade de remover, editar ou bloquear comentários e tópicos que violem este Código de Conduta.

### Como Denunciar

Se você identificar um comportamento inaceitável ou um vazamento de dados sensíveis (mesmo que acidental) no fórum, **não responda publicamente ao infrator**. 

Siga o procedimento:
1. Envie um e-mail detalhado para a equipe de moderação: `suporte@solve4me.com.br`.
2. Inclua o link direto para o comentário/tópico no GitHub Discussions.
3. Todas as denúncias são tratadas com total confidencialidade pela equipe de engenharia responsável.

### Ações Disciplinares

A depender da gravidade e reincidência da violação, a moderação aplicará as seguintes ações:

1. **Intervenção Leve (Aviso):** Remoção/Edição do conteúdo inadequado acompanhada de um aviso privado sobre a violação da regra.
2. **Suspensão Temporária:** Bloqueio dos privilégios de interação no repositório por um período determinado.
3. **Banimento Permanente:** Bloqueio definitivo do usuário na organização GitHub (aplicado imediatamente em casos de assédio grave, spam malicioso ou vazamento intencional de dados de infraestrutura).
